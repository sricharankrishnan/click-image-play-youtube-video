Introduction. What and Why Jekyll Lava Boilerplate?

Jekyll Lava Boiler Plate is a small project built to help developers by reducing that extra time in building a Jekyll Project Environment. It comes pre-built with certain features that can be useful for production level sites.

This boiler plate project is currently being served with Jekyll Gem 3.6.3. Lets get you started on how you can use this.

How To Install/Download Jekyll Lava Boilerplate?

Before downloading this boilerplate, make sure that you have ruby and jekyll pre-installed in your system. Here is a link from the Jekyll Source Page itself: https://jekyllrb.com/docs/installation/.

Then, Jekyll Lava Boiler Plate can be downloaded either directly by using the Download Button at the Github Project Repo or you could do a git clone.

Run the jekyll server and you should be able to see this exact same page on your localhost. If so then your installation can be considered successful.

git clone https://github.com/sricharankrishnan/jekyll-lava-boilerplate.git [your-project-name]
cd ./your-new-jekyll-project-directory

Getting Started with this Boilerplate

Now that you've donwloaded the boilerplate, its time to set things up before we can actually use this. It's really simple. All you have to do is remove a couple of unwanted details and you're good to go.

Please note that these details were kept in place to ensure that basic css and js requirements would come about based on this project url bring hosted on github.

Open the Layouts Directory in your project folder and open the default.html file with an editor of your choice. In this example, I am using VIM as the text editor.

vim ./_layouts/default.html

At the top of the page, locate the link tag that has the bootstrap.min.css href value. Remove the comments on either side to make this active.

{% raw %} <link href="{{ relBase }}/css/bootstrap.min.css" rel="stylesheet"> {% endraw %}

Next, proceed to the section that has the conditional rendering for stylesheets. Remove the comments for this section and make this stylesheet active as well.

{% raw %} <link href="{{ relBase }}/css/styles.css" rel="stylesheet" type="text/css"/> {% endraw %}

Remove the entire section of large comments below it along with the stylesheets that are serving the github project.

{% raw %} <!--When you are building your website, please ensure to delete this section and activate the bootstrap commented link on top and also, the conditional rendered css file for the page.bodyClass == "homePage" This was done to exclusivly serve this project on the respective github project path--> <link href="../jekyll-lava-boilerplate/css/bootstrap.min.css" rel="stylesheet"> <link href="../jekyll-lava-boilerplate/css/styles.css" rel="stylesheet" type="text/css"/> <!--Removable Stylesheet section ends here--> {% endraw %}

Activate the below scripts to enable working with the local javascript files by uncommenting them.

{% raw %} <script src="{{ relBase }}/js/bootstrap.min.js"></script>
<script src="{{ relBase }}/js/main.js" type="text/javascript"></script> {% endraw %}

And finally, you would need to remove the unwanted javascript tags that are in place that are exclusively serving this project url.

{% raw %} <!-- Please ensure to remove the script tags below. They are being used to exclusively serve js requirements from the github project directory. Otherwise you would face issues during your development process Uncomment the script tags and then run localhost with the needed jekyll command Things should be fine then. --> <script src="../jekyll-lava-boilerplate/js/bootstrap.min.js"></script> <script src="../jekyll-lava-boilerplate/js/main.js" type="text/javascript"></script> {% endraw %}

If you have followed the instructions given till this point and have understood then, start running your jekyll server and open localhost:4000 on your browser and things should be just fine. After this, run through the documenation below to get started on how things work and the features available.

jekyll serve --watch
