Introduction. What and Why Jekyll Lava Boilerplate?

Jekyll Lava Boiler Plate is a small project built to help developers by reducing that extra time in building a Jekyll Project Environment. It comes pre-built with certain features that can be useful for production level sites.

This boiler plate project is currently being served with Jekyll Gem 3.6.3. Lets get you started on how you can use this.

How To Install/Download Jekyll Lava Boilerplate?

Before downloading this boilerplate, make sure that you have ruby and jekyll pre-installed in your system. Here is a link from the Jekyll Source Page itself: https://jekyllrb.com/docs/installation/.

Then, Jekyll Lava Boiler Plate can be downloaded either directly by using the Download Button at the Github Project Repo or you could do a git clone.

git clone https://github.com/sricharankrishnan/jekyll-lava-boilerplate.git [your-project-name]
cd ./your-new-jekyll-project-directory

Run the jekyll server and you should be able to see this exact same page on your localhost. If so then your installation can be considered successful.

jekyll serve

Getting Started with this Boilerplate

Understanding this boiler plate is really simple. You would just need to have a basic understanding of certain variables that are used in this boiler plate. Knowledge of jekyll in prior is important though for you to be able to use this boiler plate. If you have downloaded this project and have things running, lets get started.

Open the Layouts Directory in your project folder and open the default.html file with an editor of your choice.

The features sections that follow describes the various directories and sections in this boiler plate to help you with your development. The layouts directory is a good place to start. If you've seen the documentation at Jekyllrb, then this should be easy to work with.

vim ./_layouts/default.html

Starter JS and SCSS file

A javascript file and a Sass File is available for you to start coding. Just erase whatever parts you don't want (except the Front Matter in the Scss file) and be on your way

The stylesheet is located at ./assets/css/styles.scss and the javascript file is located at ./assets/js/main.js.

Web Font Loader

In the ./_layouts/default.html file, you can see a section that has WebFonts.js and this font loader is responsible for loading the required fonts for this Boilerplate Project. If you would like to know how this piece works, do checkout this github project.

Otherwise, you could use the old fashion way of importing a Google Fonts API into your stylesheet or the HTML file and be on your way.
